# Petition Report/PDF Generation - Complete Specification

**Date**: February 9, 2026  
**Status**: 📋 Ready for Implementation  
**Priority**: High (Premium Feature)

---

## Overview

Allow petition creators to generate an official PDF report to present to the addressee (government official, organization, or individual). The report includes all petition data, statistics, and a verification QR code to prevent forgery.

---

## User Story

**As a petition creator**, I want to generate an official PDF report of my petition, **so that** I can present credible, verifiable evidence to the addressee and demonstrate the support my cause has received.

---

## Technical Implementation

### Database Schema

**Add to Petition Document:**

```typescript
interface Petition {
  // ... existing fields

  // Report tracking
  reportDownloads: number; // Count of downloads
  reportDownloadHistory: Array<{
    downloadedAt: Date;
    downloadedBy: string; // userId
    downloadNumber: number; // 1st, 2nd, 3rd, etc.
    paymentId?: string; // If paid download
    ipAddress?: string;
  }>;

  // Share tracking (optional - for future)
  shareStats?: {
    facebook: number;
    instagram: number;
    whatsapp: number;
    directLink: number;
    qrCode: number;
    lastUpdated: Date;
  };
}
```

### API Endpoints

**1. Generate Report**

```typescript
POST /api/petitions/[id]/report/generate

Request:
{
  petitionId: string;
}

Response:
{
  success: boolean;
  downloadUrl?: string; // If free download available
  requiresPayment?: boolean; // If needs payment
  downloadCount: number; // Current count
  freeDownloadsRemaining: number;
  message: string;
}
```

**2. Purchase Report Download**

```typescript
POST /api/petitions/[id]/report/purchase

Request:
{
  petitionId: string;
  paymentMethod: 'stripe' | 'paypal';
}

Response:
{
  success: boolean;
  paymentIntentId?: string; // For Stripe
  orderId?: string; // For PayPal
  amount: 19; // MAD
}
```

**3. Download Report**

```typescript
GET /api/petitions/[id]/report/download

Query Params:
- paymentId (optional, if paid download)

Response:
- PDF file (application/pdf)
- Filename: petition-report-{referenceCode}-{date}.pdf
```

**4. Verify Report**

```typescript
GET / api / reports / verify / [petitionId];

Response: {
  valid: boolean;
  petition: {
    title: string;
    referenceCode: string;
    createdAt: Date;
    currentSignatures: number;
  }
  reportInfo: {
    totalDownloads: number;
    lastDownloaded: Date;
  }
}
```

### PDF Generation

**Library**: `jsPDF` or `pdfkit`

**Features Needed**:

- ✅ Arabic text support (RTL)
- ✅ QR code generation
- ✅ Custom fonts (Cairo font)
- ✅ Multi-page support
- ✅ Image embedding
- ✅ Professional formatting

**Generation Process**:

1. Fetch petition data from Firestore
2. Generate QR code for verification URL
3. Create PDF with all sections
4. Add watermark with generation date
5. Store PDF temporarily (or generate on-demand)
6. Return download URL

## Payment Flow

### For 3rd+ Downloads (19 MAD)

**Step 1**: User clicks "Download Report"
**Step 2**: System checks download count
**Step 3**: If > 2, show payment modal
**Step 4**: User selects payment method
**Step 5**: Process payment (Stripe/PayPal)
**Step 6**: On success, generate PDF
**Step 7**: Trigger download
**Step 8**: Record in download history
**Step 9**: Send confirmation email
